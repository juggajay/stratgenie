# SEO Content Planner Framework - Example Topic Cluster

Real-world example applying the framework to a concrete topic cluster.

**Example Domain Context**: A compliance consulting website for strata properties

---

## Topic Cluster Definition

### Cluster Name
**AGM (Annual General Meeting) Compliance Guide**

### Primary Keyword
"AGM compliance" (Search Volume: 2,100/month, Difficulty: 42)

### Cluster Entity
**AGM Strata Compliance** (Entity type: Process/Procedure)

### Target Audience

**Primary Persona: Committee Treasurer**
- Age: 45-65
- Role: Volunteer treasurer on strata committee
- Pain Point: Unsure about AGM legal requirements
- Goal: Ensure meetings are compliant with local laws
- Tech Savvy: Low-medium
- Time Budget: Limited (volunteers, doing this part-time)

**Secondary Persona: Property Manager**
- Age: 35-50
- Role: Manages 3-5 properties
- Pain Point: Keeping up with AGM requirement changes
- Goal: Streamline AGM process without legal risk
- Tech Savvy: Medium-high
- Time Budget: Busy, needs efficient information

### Topic Cluster Architecture

```
PILLAR PAGE: "AGM Compliance: Complete Guide for Strata Committees"
├─ Keyword: AGM compliance
├─ Length: 4,000 words
├─ Scope: Overview of all compliance requirements
└─ Audience: Anyone involved in AGM process

SPOKE 1: "How to Plan an AGM: Step-by-Step Checklist"

├─ Length: 2,000 words
├─ Format: Step-by-step guide
└─ Focus: Planning and preparation phase

├─ Format: Requirements guide

├─ Keyword: AGM meeting agenda

├─ Format: Template + guide
└─ Focus: Required agenda items and structure

├─ Keyword: AGM quorum
├─ Length: 1,500 words
├─ Format: Rules explanation + examples

SPOKE 5: "AGM Minutes: Recording Requirements and Best Practices"
├─ Keyword: AGM minutes
├─ Length: 1,700 words
├─ Format: Guidelines + template
└─ Focus: Meeting documentation

SPOKE 6: "Common AGM Mistakes: Legal Pitfalls to Avoid"
├─ Keyword: AGM mistakes
├─ Length: 2,200 words
├─ Format: Cautionary guide + examples
└─ Focus: What goes wrong and why

SUPPORTING CONTENT

FAQ Page: "AGM Compliance Questions"
├─ 15 common questions from community forums
├─ Answers: 200-300 words each
├─ Keywords: Long-tail variations ("When is AGM", "Who can vote", etc.)

Glossary: "AGM Compliance Terminology"
├─ 20-25 key terms
├─ Definitions: 80-150 words each
├─ Terms: Quorum, proxy voting, ordinary resolution, special resolution, etc.

├─ Key differences in requirements
├─ Links to specific spoke articles for details

Case Study: "How ABC Apartments Improved AGM Compliance"
├─ Before/after story
├─ Specific changes made
├─ Results and metrics
├─ Downloadable checklist
```

- **Commercial Investigation**: 30% (Comparison, mistake article, case study)

## BASIC INFORMATION
- Content Type: Pillar Page (Cluster Hub)
- Cluster: AGM Compliance
- Target Publication: End of Month 1, Week 2
- Primary Author: Senior Writer (Expert-level required)
- Reviewer: Content Editor + Legal Reviewer

## TARGET KEYWORDS
- Primary: agm compliance (2,100/mo, 42 difficulty)
- Secondary:

## SEARCH INTENT
- Intent Type: Informational (Primary), Commercial Investigation (Secondary)
- User Stage: Awareness + early consideration
- User Needs:
  - Understand what AGM compliance means
  - Learn all requirements they must follow
  - Get overview of entire process
  - Avoid legal issues
  - Know where to find detailed guidance

## CONTENT OUTLINE

### H1: AGM Compliance: Complete Guide for Strata Committees
(Include primary keyword naturally)

### Introduction (300-400 words)
- Hook: Why AGM compliance matters (liability, legal requirements)

- Overview: What this guide covers (all subtopics at high level)

- Define AGM and compliance requirements

  - Minutes requirements

### H2: Common Compliance Mistakes to Avoid (400-500 words)

## REQUIREMENTS

### Content Specifications
- Target Word Count: 4,000 words (range: 3,800-4,200)
- Reading Level: Grade 8-10 (accessible to non-experts)
- Format: Scannable with short paragraphs, bullet points, subheadings
- Sections: 7-9 main H2 sections, 15-20 H3 subsections

### Keyword Integration
- Primary Keyword Density: 0.9-1.2% (target: 36-48 mentions)
- Secondary Keywords: "agm requirements" (4-6x), "agm checklist" (3-5x), others naturally
- LSI Keywords: compliance, annual meeting, notice, agenda, quorum, minutes, etc.
- Entity Mentions: Strata Act [jurisdiction], specific regulations, local requirements

### Internal Links
**Mandatory**: Back links in future spokes (they'll link here)
**Inbound Links**:
- Spoke 1: Plan an AGM (1 link)
- Spoke 2: Notice Requirements (1 link)
- Spoke 3: Meeting Agenda (1 link)
- Spoke 4: Quorum Rules (1 link)
- Spoke 5: Minutes (1 link)
- Spoke 6: Mistakes (1 link)
- FAQ page (1 link)
- Glossary (1 link)
**Total**: 8-10 internal links with contextual anchor text

### Visual Elements
- 1-2 infographics (timeline, process flowchart)
- 1-2 comparison tables (requirements by stage)
- Icon graphics for checklist items (optional)

### E-E-A-T Requirements
**Experience**: Include example scenarios from real strata meetings
**Expertise**: Reference to actual strata laws/regulations (cite them)
**Authority**: Quote from industry expert, reference to professional association
**Trust**: Disclaimer about needing legal advice for specific situations

### Metadata
- Meta Description (draft): "Complete AGM compliance guide for strata committees. Learn all requirements, avoid common mistakes, and ensure legal meetings. [120-130 chars]"
- Focus Keyword: agm compliance
- Canonical: [domain]/agm-compliance-guide/

## Key Success Factors for This Cluster

1. **Pillar-First Approach**: Pillar must publish before spokes for context
2. **Legal Accuracy**: Cite actual regulations (high E-E-A-T importance)
3. **Practical Focus**: Include checklists/templates (users need actionable content)
4. **Clear Navigation**: Hub-spoke structure makes it easy to find detailed info
5. **Consistent Terminology**: Use glossary to ensure term consistency
6. **Timeline Awareness**: AGM season drives search volume patterns

## Lessons Learned for Next Cluster

- [ ] Document how long each article type actually takes (vs. estimate)
- [ ] Track which spokes drive most internal traffic (repeat success)
- [ ] Monitor which content types perform best (pillar vs. case study vs. FAQ)
- [ ] Note any cannibalization issues (even though cluster is new)
- [ ] Assess E-E-A-T adequacy based on ranking success
- [ ] Plan backlink strategy earlier (starts with pillar publication)
