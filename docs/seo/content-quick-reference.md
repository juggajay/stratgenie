## 5 Hook Templates

### 1. Problem-Solution
```
[Problem] + [Promise] + [Context]
```
Example: "Running your first AGM can feel daunting. This guide breaks down the process into manageable steps. By the end, you'll have a clear roadmap."

### 2. Authority-Urgency
```
[New requirement] + [Why it matters] + [What guide provides]
```
Example: "Strata Hub registration is now mandatory. Failure to register can result in penalties of up to $2,200. This guide walks you through everything."

### 3. Confusion-Clarity
```
[Common confusion] + [Why confusing] + [Promise of clarity]
```
Example: "The difference between admin and capital funds confuses many owners. While both are required, they serve different purposes. This guide explains everything."

### 4. Statistics-Stakes
```
[Statistic] + [What it means] + [What guide covers]
```
Example: "60% of schemes have underfunded capital funds, risking special levies. Understanding proper fund management is crucial. This guide covers calculations to pitfalls."

### 5. Empowerment
```
[Current state] + [Transformation] + [How delivered]
```
Example: "The 2025 reforms emphasize committee accountability. Understanding your duties protects you from liability. This guide breaks down requirements."

## 4 Body Structure Patterns

### A. Comprehensive Guide (2,000-3,000 words)
```
H1: Title
[Hook]
H2: Understanding [Topic] (context)
H2: Before You Start (prep)
H2: Step-by-Step Process
  H3: Step 1
  H3: Step 2
  [...]
H2: Common Mistakes
H2: Compliance/Requirements
H2: How StrataGenie Helps [CTA]
H2: FAQs
[Legal disclaimer]
```

### B. Explainer/Comparison (1,500-2,000 words)
```
H1: Title
[Hook]
H2: Overview
H2: [Concept A] Explained
  H3: What It Covers
  H3: How It Works
H2: [Concept B] Explained
  H3: What It Covers
  H3: How It Works
H2: Key Differences (table)
H2: Common Mistakes
H2: Practical Examples
H2: How StrataGenie Helps [CTA]
H2: FAQs
[Legal disclaimer]
```

### C. Compliance Update (1,200-1,800 words)
```
H1: Title
[Hook]
H2: What's Changed?
H2: Key Requirements
  H3: Requirement 1
  H3: Requirement 2
H2: How to Comply
H2: Consequences of Non-Compliance
H2: FAQs
H2: Next Steps [CTA]
[Legal disclaimer]
```

### D. Problem-Solution (1,000-1,500 words)
```
H1: Title
[Hook]
H2: Understanding the Problem
H2: Who This Affects
H2: Solution Overview
H2: Step-by-Step Solution
  H3: Immediate Actions
  H3: Short-term Fixes
  H3: Long-term Prevention
H2: How StrataGenie Helps [CTA]
H2: Related Resources
[Legal disclaimer]
```

## Title Formulas

### Formula 1: Complete Guide
```
Complete Guide to [Topic] in [Location] ([Year])
```

### Formula 2: How-To
```
How to [Action]: [Benefit] for [Audience]
```

### Formula 3: Comparison
```
[A] vs [B]: Understanding [Topic] in [Context]
```

### Formula 4: Update/News
```
[Topic]: New [Change] Rules for [Year]
```

**Rules:**
- 50-60 characters
- Primary keyword near beginning
- Include location (NSW) for local SEO
- Include year for time-sensitive content

## Meta Description Formula

```
[Benefit with keyword]. [What reader learns]. [Secondary benefit/authority].
```

**Rules:**
- 150-160 characters
- Include primary keyword
- Active voice
- End with period

**Example:**
> "Step-by-step guide to registering your strata scheme on NSW Strata Hub. Learn what information you need, common mistakes to avoid, and how to stay compliant."

## MDX Callout Components

### Tip (Helpful advice)
```mdx
<Tip>
Online meetings can significantly improve attendance.
</Tip>
```

### Warning (Caution)
```mdx
<Warning>
Failing to give proper notice can invalidate the meeting.
</Warning>
```

### Danger (Serious consequences)
```mdx
<Danger>
Invalid AGMs can lead to legal challenges and compliance issues.
</Danger>
```

### Info (Factual information)
```mdx
<Info>
Section 93 of the Act requires these statements in prescribed form.
</Info>
```

### Legal Note (Disclaimer)
```mdx
<LegalNote>
This guide is for informational purposes only. Consult NSW Fair Trading.
</LegalNote>
```

### CTA (Call to action)
```mdx
<TrialCTA />
```

## Writing Rules Checklist

### Voice & Tone
- [ ] Professional but approachable
- [ ] Plain English (no jargon without explanation)
- [ ] Active voice
- [ ] Second person ("you") for instructions
- [ ] Confident and authoritative

### Sentence Structure
- [ ] Average sentence: 15-20 words
- [ ] Max sentence: 25 words
- [ ] Varied sentence lengths
- [ ] One idea per sentence

### Paragraph Structure
- [ ] 2-3 sentences ideal
- [ ] Max 4 sentences
- [ ] One topic per paragraph
- [ ] Transition words between paragraphs

### Formatting
- [ ] **Bold** for key concepts and warnings
- [ ] Lists for multiple items (not paragraphs)
- [ ] Tables for comparisons
- [ ] Code formatting for references
- [ ] Callout boxes for important info

### Headings
- [ ] Single H1 (title only)
- [ ] 4-8 H2 sections
- [ ] H3s to break up long H2 sections
- [ ] Keywords in headings
- [ ] Descriptive, not clever

## Internal Linking Strategy

**Requirements per article:**
- Minimum: 3 internal links
- Maximum: 8 internal links
- Link to pillar content
- Link to related guides
- Descriptive anchor text

**Good anchor text:**
```markdown
[running your first AGM](/blog/how-to-run-agm-self-managed-strata)
```

**Bad anchor text:**
```markdown
[click here](/blog/some-article)
```

## FAQ Section Guidelines

**Number of questions:** 4-6

**Question format:**
```markdown
### [Question with keyword if natural]?

[Clear, concise answer. 2-3 sentences. Actionable if possible.]
```

**FAQ rules:**
- Use actual user questions
- Answer directly (no fluff)
- 50-100 words per answer
- Include relevant keywords
- Link to related content if helpful

## Common Mistakes to Avoid

### Writing
- ❌ Long, dense paragraphs
- ❌ Jargon without explanation
- ❌ Passive voice overuse
- ❌ Unclear next steps

### SEO
- ❌ Keyword stuffing
- ❌ Missing meta description
- ❌ No internal links
- ❌ Generic anchor text ("click here")

### Structure
- ❌ Missing H2 sections
- ❌ Too many H3s (use H2 instead)
- ❌ No logical flow
- ❌ Skipping FAQ section

### Product
- ❌ Forcing product mentions
- ❌ Generic marketing speak
- ❌ Multiple CTAs
- ❌ CTA in introduction

## URL Structure

**Formula:**
```
/blog/[slug]
```

**Slug rules:**
- Primary keyword
- Lowercase
- Hyphens (not underscores)
- 3-5 words
- No stop words
- No year (use in title)

**Good:**
- ✅ `/blog/strata-hub-registration-guide`
- ✅ `/blog/admin-fund-vs-capital-works-fund`

**Bad:**
- ❌ `/blog/a-complete-guide-to-strata-hub-2025`
- ❌ `/blog/article_12345`

## Content Type Decision Tree

```
Is it a complete process?
  YES → Pattern A (Comprehensive Guide)
  NO  ↓

Is it explaining/comparing concepts?
  YES → Pattern B (Explainer/Comparison)
  NO  ↓

Is it a new law/regulation?
  YES → Pattern C (Compliance Update)
  NO  ↓

Is it solving a specific problem?
  YES → Pattern D (Problem-Solution)
  NO  → Reconsider topic
```

## Quick Content Audit

**Good content has:**
- ✅ Clear structure with scannable headings
- ✅ Actionable advice and next steps
- ✅ Real examples and scenarios
- ✅ Visual elements (tables, callouts)
- ✅ Accurate, up-to-date information
- ✅ Natural product integration
- ✅ Complete FAQ section

**Bad content has:**
- ❌ Wall of text with no breaks
- ❌ Vague, generic advice
- ❌ No examples or scenarios
- ❌ All text, no visual variety
- ❌ Outdated or questionable info
- ❌ Forced, salesy product mentions
- ❌ No FAQ or incomplete answers
